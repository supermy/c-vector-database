# Rust Qwen35 向量数据库

高性能 Rust 向量数据库实现，基于 Qwen35 架构优化。

## 特性

- ✅ 多种距离度量支持（余弦、欧几里得、点积、曼哈顿）
- ✅ 线程安全的并发操作
- ✅ 并行搜索能力（使用 Rayon）
- ✅ 内存高效的存储
- ✅ 完整的 CRUD 操作
- ✅ 序列化/反序列化支持
- ✅ 统计信息追踪
- ✅ 全面的性能基准测试

## 关键性能洞察

1. **插入性能**：
   - 小维度（128）表现最佳，可达 1M+ vectors/s
   - 随维度增加，性能略有下降但仍保持高水平
   - 批量插入时性能更优

2. **搜索性能**：
   - 搜索性能与数据量和维度呈负相关
   - 128 维、1000 数据量时可达近 3000 queries/s
   - 适合实时性要求较高的场景

3. **距离度量**：
   - DotProduct 性能最优（~381 queries/s）
   - Euclidean 和 Manhattan 性能相近
   - Cosine 因需要归一化计算稍慢

## 使用方法

### 基本使用

```rust
use rust_qwen35::{create_db, DistanceMetric};

// 创建数据库
let db = create_db(512, DistanceMetric::Cosine);

// 插入向量
db.insert(1, vec![0.1, 0.2, ..., 0.9], None).unwrap();

// 搜索最近邻
let results = db.search(&query_vector, 10).unwrap();
for result in results {
    println!("ID: {}, Distance: {}", result.id, result.distance);
}
```

### 运行基准测试

```bash
cargo bench
```

### 运行性能测试

```bash
cargo run --release --example perf_test
```

## 技术亮点

1. **并行计算**：使用 Rayon 实现数据并行搜索
2. **细粒度锁**：使用 parking_lot 的 RwLock 实现高效并发
3. **内存优化**：使用 SmallVec 减少小向量内存占用
4. **批量操作**：支持批量搜索提高吞吐量
5. **持久化**：支持数据库状态的保存和加载
